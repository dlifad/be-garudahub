ALUR SINGKAT ADMIN (PLAYER -> TOURNAMENT -> MATCH -> LINEUP)
1. Siapkan data master pemain di players.
2. Assign pemain ke turnamen melalui tournament_players.
3. Buat match untuk turnamen terkait.
4. Saat set lineup match, pemain harus berasal dari squad turnamen match tersebut.
5. Simpan pemain ke match_lineups via endpoint lineup.
6. Detail match menampilkan lineup sesuai data pada match tersebut.

CATATAN PENTING (SEED DATA)
- players.json adalah master pemain dan sekarang menyimpan player_code sebagai identifier stabil lintas environment.
- tournament_players.json sekarang memakai format relasi: tournament_id + player_code.
- Seeder tournament_players memproses urutan fallback: player_code -> player_id -> player_name (untuk kompatibilitas data lama).
- tournament_coaches.json menyimpan riwayat coach per turnamen dan sudah disiapkan untuk seeder backend.
- matches.json sekarang sudah menyimpan formation per match.
- match_lineups.json sekarang menyimpan lineup starter/subs per match lengkap dengan is_captain dan posisi detail (GK/CB/LCB/RCB/RB/LB/RWB/LWB/CM/CDM/CAM/LM/RM/LW/RW/ST).
- Seeder matches sudah mode upsert dengan konflik unik (tournament_id, match_date_utc), sehingga aman dijalankan berulang (idempotent) tanpa duplikasi match.
- Seeder match_lineups sudah mode upsert, resolve pemain via player_code, dan resolve match via kombinasi match_tournament_id + match_date_utc.
- Backend tetap memvalidasi bahwa pemain harus terdaftar di squad turnamen sebelum bisa masuk lineup match.
